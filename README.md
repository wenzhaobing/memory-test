## 一、产品概述

### 1.1 产品定位
《童年的碎片》是一款**碎片化叙事 + 情感治愈**的文字类手机游戏。玩家通过回答关于3岁前模糊记忆的选择题，逐步“拾起”记忆碎片，最终生成一份专属的**童年画像**和**自动生成的信**。

### 1.2 核心理念
> **没有对错，只是与小时候的自己重逢。**

- 不评判玩家的选择
- 不制造焦虑或竞争
- 重点是“被看见”和“被理解”

### 1.3 目标用户
- 年龄：18-35岁
- 特征：对自我探索有兴趣、喜欢治愈向内容、有碎片化时间
- 场景：通勤、睡前、休息间隙

## 二、核心玩法机制

### 2.1 整体流程

```
引导页（仅首次）
      │
      ▼
碎片拾忆（10个碎片）
      │
      ├── 温度计实时变化（5档）
      ├── 里程碑触发（3/6/9片）
      └── 后台累积4个维度分数
      │
      ▼
画像生成 + 自动写信
      │
      ├── 画像名称 + 一句话标签
      ├── 4个维度进度条
      └── 信封（点击展开信）
      │
      ▼
结局（分享 / 重拾记忆）
```

### 2.2 碎片选择机制

**每个碎片包含**：
- 标题（如“外婆的手”）
- 描述文字（斜体+下划线，手帐风格）
- 2-3个选项

**选项结构**：
| 属性 | 说明 | 示例 |
|------|------|------|
| 选项文本 | 展示给玩家 | “我记得那种触感” |
| 温度delta | 影响温度计 | +1 / 0 / -1 |
| 依恋影响 | 影响依恋维度 | 亲密/安全/焦虑/疏离 |
| 韧性影响 | 影响韧性维度 | 有力量/坚强/脆弱 |
| 清晰度影响 | 影响清晰度维度 | +1 / +0.5 / 0 |
| 反馈文案 | 选择后弹窗显示 | “长大了以后，再也没有人那样捏我的手。” |

## 四、核心组件设计

### 4.1 温度计（5档独立配色）

**位置**：场景2顶部

**设计**：
- 左侧：当前档位图标（❄️🌥️🌱☀️🔥）
- 右侧：5段进度条 + 当前档位描述
- 进度条每档独立颜色（从冷到暖渐变）

**档位映射**：

| 档位 | 图标 | 名称 | 进度条颜色 | 描述 |
|------|------|------|-----------|------|
| 1 | ❄️ | 冰封 | `#8F9CA8` | 你不想说话 · 像被冻住了 |
| 2 | 🌥️ | 微凉 | `#9CB0A0` | 心里闷闷的 · 像要下雨还没下 |
| 3 | 🌱 | 常温 | `#E8B06E` | 今天还好 · 心里软软的 |
| 4 | ☀️ | 温暖 | `#D48C6B` | 心里软软的 · 好像被抱了一下 |
| 5 | 🔥 | 燃烧 | `#C46A4A` | 想哭，也想笑 · 被理解了 |

### 4.2 碎片卡片

**结构**：
- 头部：标题（图标 + 文字）
- 主体：描述文字（斜体+下划线+背景色）
- 底部：选项按钮列表

**选项交互**：
- 点击选项 → 显示轻弹窗反馈（1.5秒）
- 弹窗颜色根据delta变化：升温（暖橘）/降温（灰绿）/不变（米色）

### 4.3 里程碑弹窗

**触发条件**：完成第3、6、9个碎片时

**设计**：
- 居中弹窗，半透明遮罩背景
- 图标 + 标题 + 引用文案
- 停留1.8秒，点击任意位置关闭

**内容配置**：

| 里程碑 | 图标 | 标题 | 引用文案 |
|--------|------|------|---------|
| 第3片 | 🌼 | 第三片记忆 | “你开始看见小时候的轮廓了” |
| 第6片 | 🌼 | 第六片记忆 | “你越来越靠近那个小孩了” |
| 第9片 | 🌾 | 第九片记忆 | “就差最后一片了。你突然不那么着急了。” |

### 4.4 画像卡片

**结构**：
- 画像图标 + 名称 + 一句话标签
- 虚线分割线
- 4个维度进度条

**画像类型（示例）**：

| 画像名称 | 图标 | 标签 | 触发条件 |
|---------|------|------|---------|
| 热烈的连接者 | 🔥 | “你小时候就像一团火，敢爱敢痛” | 温度≥4 + 依恋≥60% + 韧性≥60% |
| 温暖的守护者 | ☀️ | “你默默地守护着很多事情” | 温度≥3 + 依恋≥50% + 韧性≥50% |
| 安静的观察者 | 🌙 | “你藏在角落，但从来没有真的离开” | 温度≤2 + 依恋≤40% + 韧性≤40% |

### 4.5 信封与信

**信封样式**：
- 仿真实信封：封口、地址线、邮票位
- 文案：“TO · 小时候的我” + “一封迟到的信”

**交互**：
- 点击信封 → 信封消失 → 信纸展开（带动画）
- 信的内容自动生成（基于画像和倾向）
- 信头固定：“亲爱的小时候的我：”
- 信尾固定：“—— 现在的我”

**信纸样式**：
- 米白背景 + 浅边框
- 标题居中 + 虚线分割
- 正文字体：Georgia 斜体

### 4.6 按钮

| 按钮 | 样式 | 功能 |
|------|------|------|
| 分享画像 | 暖橘背景 | 生成分享图片/链接 |
| 重拾记忆 | 米色背景 | 清空数据，重新开始 |

## 五、四个维度定义与计算

### 5.1 维度定义

| 维度 | 图标 | 含义 | 取值范围 |
|------|------|------|---------|
| 温度 | 🌡️ | 情绪的冷暖 | 1-5档 → 0-100% |
| 依恋 | 🤝 | 与他人的情感连接 | 0-100% |
| 韧性 | 🌱 | 自我恢复能力 | 0-100% |
| 清晰度 | 🌫️ | 记忆的清晰程度 | 0-100% |

### 5.2 计分规则

**温度**：
- 初始：档位3（常温）
- 每次选择：+1（升温）/ 0 / -1（降温）
- 最终：档位 → 百分比

**依恋**：
- 初始：0分
- 亲密选项：+1，安全选项：+0.5
- 焦虑选项：-0.5，疏离选项：-1
- 最终：(分数+6)×7+10 → 限制0-100%

**韧性**：
- 初始：0分
- 有力量选项：+1，坚强选项：+0.5
- 脆弱选项：-0.5
- 最终：(分数+5)×8+15 → 限制0-100%

**清晰度**：
- 初始：0分
- “我记得”：+1，“模糊记得”：+0.5，“不记得”：0
- 最终：分数×7+5 → 限制0-100%

## 六、场景定义（4个场景）

### 场景1：首次引导
- 触发：首次打开游戏
- 存储：localStorage记录，不再重复显示
- 内容：欢迎语 + 玩法说明 + “开始拾忆”按钮

### 场景2：碎片拾忆（核心玩法）
- 内容：温度计 + 碎片卡片列表
- 交互：点击选项 → 温度变化 + 弹窗反馈 + 后台计分

### 场景3：里程碑弹窗
- 触发：完成第3、6、9个碎片时
- 内容：居中弹窗，停留1.8秒

### 场景4：画像与信（结局）
- 内容：画像卡片 + 信封 + 信纸
- 交互：点击信封展开信纸
- 按钮：分享画像、重拾记忆

## 七、技术规范

### 7.1 平台
- 优先：手机Web（iOS/Android浏览器）
- 后续：可打包为小程序或App

### 7.2 存储
- localStorage：存储游戏进度、已选选项、维度分数
- 无需后端（MVP阶段）

### 7.3 性能
- 首屏加载 < 2秒
- 动画帧率 ≥ 60fps
- 离线可用（一次加载后）
